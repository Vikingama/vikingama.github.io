《CSS 选择器世界》笔记

最近在读张鑫旭的 CSS 三部曲，这是第二本，主要讲 CSS 选择器，简单做下笔记

Jan 20, 2022
阅读时长：7 分钟

优先级与选择器

CSS 是有局部作用域的（ShadowDOM）。

CSS 选择符有五种：

  1. 后代关系（空格） ；
  2. 父子关系（>） ；
  3. 相邻兄弟关系（+） ；
  4. 随后兄弟关系（~） ；
  5. 列关系（||） 。

CSS 优先级有六种：

  1. 0 级：通配选择器（*，不包括伪元素）、选择符（空格、>、+、~、||）和逻辑组合伪类；
  2. 1 级：标签选择器；
  3. 2 级：类选择器、属性选择器和伪类；
  4. 3 级：ID 选择器；
  5. 4 级：内联样式；
  6. 5 级：!important（可以使 JS 无效）。

CSS 选择器优先级与 DOM 位置无关。

<style>
  /* 选择器的优先级并不考虑 DOM 的位置，所以背景是绿色 */
  div p {
    background: red;
  }
  html p {
    background: green;
  }
  /* 提升选择器优先级的方法：重复选择器自身 */
  .paragraph.paragraph {
    font-size: 24px;
  }
  /* 提升选择器优先级的方法：使用属性选择器 */
  .paragraph[id] {
    color: red;
  }
  .paragraph {
    color: black;
    font-size: 12px;
  }
</style>
<div>
  <p class="paragraph" id="paragraph">
    背景是 green，字体颜色是 red，字体大小是 24px。
  </p>
</div>

在 HTML 中，标签和属性都是不区分大小写的，而属性值是区分大小写的。于是，在 CSS 中，标签选择器不区分大小写，属性选择器中的属性也不区分大小写，而类选择器和 ID 选择器本质上是属性值，因此要区分大小写。

  </p>
</div>

CSS 选择器最佳实践：

  1. 不要使用 ID 选择器，优先级太高，使得样式覆盖变得困难；
  2. 样式重置可以使用标签选择器或属性选择器；
  3. 不要嵌套选择器：
    1. 渲染性能差，CSS 是从右向左匹配渲染的（.box > div 是先匹配所有的 div 再匹配 box 类名的）；
    2. 选择器性能排序：ID 选择器>类选择器>标签选择器>通配选择器>属性选择器>伪类选择器；
    3. 每多嵌套一层选择器就会多一层匹配和计算；
    4. 优先级混乱，嵌套越多，优先级计算越复杂；
    5. 样式布局脆弱，过多的层级会和 HTML 层级结构进行绑定，不好维护。
  4. 命名建议使用小写，使用英文或缩写，专有名词可以使用拼音；
  5. 命名设置统一前缀，强化品牌的同时避免样式冲突；
  6. 对于组合命名，可以使用短横线或下划线。

为什么没有前面兄弟选择器？

受制于 DOM 渲染规则；如果 CSS 支持了前面兄弟选择器或者父元素选择器，那就必须要等页面所有子元素加载完毕才能渲染 HTML 文档。网页呈现速度必然会大大减慢，浏览器会出现长时间的白板，这会造成不好的体验。

属性选择器

八种属性选择器：

  1. [attr]，只要包含指定的属性就匹配；
  2. [attr=“val”]，属性值完全匹配选择器；
  3. [attr~=“val”]，属性值单词完全匹配选择器；
  4. [attr|=“val”]，属性值起始片段完全匹配选择器（val、val-*）；
  5. [attr^=“val”]，属性值以字符 val 开头的元素；
  6. [attr$=“val”]，属性值以字符 val 结尾的元素；
  7. [attr*=“val”]，属性值包含字符 val 的元素；
  8. [attr=“val” i]，以上几种均适用，匹配时忽略属性值大小写。

  console.log(document.querySelectorAll('[data-id]')); // NodeList(4) [p.a, p.b, p.c, p.d]
  console.log(document.querySelectorAll("[data-id='one']")); // NodeList [p.b]

伪类

:focus 默认只能匹配特定的元素：

  1. 设置了 contenteditable 属性的普通元素；
  2. 设置了 tabindex 属性的普通元素
  3. 非 disabled 状态的表单元素；
  4. 包含 href 属性的 a 元素；
  5. area、summary 元素。

:focus 只有在当前元素处于聚焦状态的时候才匹配，而 :focus-within 在当前元素或者是当前元素的任意子元素处于聚焦状态的时候都会匹配。换句话说，子元素聚焦，可以让父级元素的样式发生变化。:focus-within 的行为本质上是一种“父选择器”行为，子元素的状态会影响父元素的样式。

:focus-visible 可以让我们知道元素的聚焦行为到底是鼠标触发还是键盘触发；如果希望去除鼠标点击时候的 outline，而保留键盘访问时候的 outline，只要一条 CSS 规则就可以了：

1
2
3

:focus:not(:focus-visible) {
  outline: 0;
}

:placeholder-shown 表示当输入框的 placeholder 内容显示的时候，匹配该输入框；通过此伪类可以实现比较有意思的 交互 。

:checked 与 [checked] 的区别：

  1. :checked 只能匹配标准表单元素，不能匹配普通元素；
  2. [checked] 可以与设置了 checked 属性的任意元素匹配；
  3. [checked] 不会响应 JS 对元素属性的修改；
  4. :checked 可以匹配从祖先元素那里继承过来的状态。

可以对输入值进行验证的伪类：

  1. :valid 与 :invalid；
  2. :in-range 与 :out-of-range；
  3. :required 与 :optional；
  4. :user-invalid；
  5. :blank。

:root 表示的就是 html 元素，只是 :root 优先级更高。对应的，在 ShadowDOM 中使用 :host 伪类。

:empty 的使用：

  1. 不会匹配含有注释/空格的空元素（毕竟注释和空格都算是节点）；
  2. 不会匹配没有闭合的闭合元素；
  3. ::before/::after 不会影响匹配；
  4. 可以非常方便的隐藏空节点；
  5. 结合伪元素，可以在空元素内做一些用户提示。

否定伪类 :not()：

  1. 本身的优先级为 0，最终选择器优先级是由括号里的表达式决定的，:not(p) 的优先级等同于标签选择器的优先级；
  2. :not() 可以级联，例如，input:not(:disabled):not(:read-only)；
  3. 不支持多个表达式，也不支持选择符，例如，:not(li, dd) 是无效的。

:fullscreen 伪类用来匹配全屏元素；通过调用 element.requestFullScreen 可以让元素全屏显示，通过调用 document.cancelFullScreen 可以取消全屏。

:lang() 用来匹配指定语言环境下的元素。
