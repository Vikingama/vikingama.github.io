Renderless Component

2020.04.29

昨天面试被问到了 Renderless 组件，一脸懵逼。

具名插槽

具名插槽相当于一个占位符，它的内容会被父组件插入的内容所取代：

<!-- Child.vue -->
<template>
  <div class="child">
    <slot name="namedSlot"> Named Slot Content </slot>
    <slot> Default Slot Content </slot>
  </div>
</template>
<!-- Parent.vue -->
<template>
  <div class="parent">
    <Child>
      <h1>出现在默认插槽里</h1>
      <template #namedSlot>
        <h2>出现在具名插槽里</h2>
      </template>
      <h1>出现在默认插槽里</h1>
    </Child>
  </div>
</template>
<!-- 渲染结果 -->
<div class="parent">
  <div class="child">
    <h2>出现在具名插槽里</h2>
    <h1>出现在默认插槽里</h1>
    <h3>出现在默认插槽里</h3>
  </div>
</div>

作用域插槽

作用域插槽就是给普通插槽添加了传递参数的能力；作用域插槽会被编译为一个返回 VNode 的函数，通过函数调用，可以将子组件的变量传递到父组件；很像 React 的 Render Props 模式：

<!-- Child.vue -->
<template>
  <ul class="child">
    <li v-for="item in list" :key="item.name">
      <slot name="item" :itemData="item"> 我是候补... </slot>
    </li>
  </ul>
</template>
<!-- Parent.vue -->
<template>
  <div class="parent">
    <Child>
      <template #item="{itemData}">
        <a href="itemData.link">{{ itemData.name }}</a>
      </template>
    </Child>
  </div>
</template>
<!-- 渲染结果 -->
<div class="parent">
  <ul class="child">
    <li><a href="itemData.link">Baidu</a></li>
    <li><a href="itemData.link">Google</a></li>
    <li><a href="itemData.link">So</a></li>
  </ul>
</div>

Renderless 组件

Renderless 组件不会渲染具体内容，它仅仅管理一些状态和行为，将这些状态和行为通过作用域插槽暴露给父组件，由父组件对渲染内容进行控制：

<!-- RenderlessComponent.vue -->
<script>
  export default {
    render() {
      // 通过 $slots[name] 访问具名插槽，通过 $scopedSlots[name] 访问作用域插槽
      return this.$scopedSlots.default({
        value: 'ABCer'
      });
    }
  };
</script>
<!-- Parent.vue -->
<div class="parent">
  <renderless-component>
    <template #default="{value}">
      <p>Hello, {{value}}</p>
    </template>
  </renderless-component>
</div>
<!-- 渲染结果 -->
<div class="parent">
  <p>Hello, ABCer</p>
</div>

举个别人的例子：

<!-- RenderlessComponent.vue -->
<script>
  export default {
    props: ['value'],
    data() {
      return {
        tag: ''
      };
    },
    methods: {
      add() {
        if (!this.tag || this.tag.trim().length === 0) {
          return;
        }
        this.$emit('input', [...this.value, this.tag.trim()]);
        this.tag = '';
      },
      remove(tag) {
        this.$emit(
          'input',
          this.value.filter((v) => v !== tag)
        );
      }
    },
    render() {
      return this.$scopedSlots.default({
        list: this.value,
        add: this.add, // 传了不一定会用，只是为了适应多种业务场景
        remove: this.remove,
        inputAttrs: {
          value: this.tag
        },
        inputEvents: {
          input: (e) => {
            this.tag = e.target.value;
          },
          keydown: (e) => {
            if (e.keyCode === 13) {
              e.preventDefault();
              this.add();
            }
          }
        }
      });
    }
  };
</script>
<!-- Parent.vue -->
<template>
  <renderless-component v-model="tags">
    <template #default="{list, remove, inputAttrs, inputEvents}">
      <div class="tags-input">
        <span class="tags-input-tag">
          <span v-for="item in list" :key="item"> {{ item }} </span>
          <button class="tags-input-remove" @click="remove">&times;</button>
        </span>
        <input
          class="tags-input-text"
          placeholder="add tag"
          v-bind="inputAttrs"
          v-on="inputEvents"
        />
      </div>
    </template>
  </renderless-component>
</template>
<script>
  export default {
    data() {
      return {
        tags: ['Testing', 'Design']
      };
    }
  };
</script>

当项目中有很多逻辑层相同展示层不同的组件时，Renderless 组件可以很好地实现逻辑复用。
